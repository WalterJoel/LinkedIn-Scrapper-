- PARA MANEJAR DISTINTAS VERSIOENS DE NODE USAMOS NVM QUE PERMITE SALTAR DE VERSION EN VERSION CUANDO SE REQUIERA
- yarn init -y
- git ignore, agrego que cosas no quiero que suban al github
- esbuild es una dependencia de desarrollo para eso uso --dev (yarn add esbuild --dev)
- CUANDO UTILIZO EL GITIGNORE USO POR EJEMPLO dist/, ese slash significa que ignoro recursivamente todo lo que viene
- BUNDLE EN EL PACKAGE.JSON, GENERA MAS CODIGO EN EL DIST
- MINIFY, PARA REDUCIR CODIGO EN UNA SOLA LINEA
- MIMIFICO PARA QUE CORRA MAS RAPIDO O NADIE ENTIENDA MI CODIGO, YA QUE CAMBIA LOS NOMBRES DE LAS CLASES, POR LETRAS COMO 'O', ETC
